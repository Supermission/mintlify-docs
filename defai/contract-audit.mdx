# Check if a contract is safe before you interact

The **Contract Audit** agent analyzes smart contracts for vulnerabilities, access control issues, and security best practices. Use it from the DeFAI chat or the dedicated Security page.

## How to use

**From DeFAI chat:**
- `"Audit this contract: 0x..."`
- `"Check for reentrancy vulnerabilities in this token contract"`
- `"Is this contract safe to interact with?"`

**From the Security page:**
- Dedicated interface focused on contract auditing
- Prompt cards to get started: "Audit a contract by address", "Check for vulnerabilities", "Gas optimization review", "Standards compliance check"
- **Deep audit toggle** — extended analysis for more thorough vulnerability detection
- Session history so you can revisit past audits

## Risk score

A **circular score gauge** (0-100) with color coding:
- **0-25**: Low risk (green)
- **25-50**: Medium risk (amber)
- **50-75**: High risk (orange)
- **75-100**: Critical risk (red)

Overall risk level banner: `SAFE`, `CAUTION`, `WARNING`, or `AVOID`.

## Findings

Each finding is a severity-coded card:

| Severity | Color | Description |
|----------|-------|-------------|
| **CRITICAL** | Red | Exploitable vulnerabilities, fund loss risk |
| **HIGH** | Orange | Significant security concerns |
| **MEDIUM** | Amber | Moderate issues that should be addressed |
| **LOW** | Blue | Minor issues, best practice violations |
| **INFO** | Gray | Informational notes, no security impact |

Each finding card contains:
- **Title** — concise description of the issue
- **Description** — detailed explanation of the vulnerability
- **Code snippet** — relevant code with line numbers
- **Recommendation** — how to fix or mitigate the issue

## Deep audit

When you enable the deep audit toggle, the system runs an extended analysis with advanced vulnerability detection. This takes longer but catches patterns that the standard audit might miss — especially useful for contracts you're about to trust with your funds.

## When to use it

- **Before approving a new protocol** — check the contract before you grant token approvals.
- **Before depositing into a yield opportunity** — make sure the vault contract is solid.
- **Researching a new token** — check the token contract for minting backdoors or owner privileges.
- **Due diligence on a DeFi project** — audit their core contracts before committing capital.

<Warning>
  AI-powered audits are a supplement to, not a replacement for, professional security audits. For contracts handling significant value, always commission a formal audit from a reputable security firm. This tool is best for initial screening and quick assessments.
</Warning>

<Tip>
  Use the deep audit for contracts you're about to interact with for the first time, especially if they handle deposits or approvals. The extended analysis catches patterns that the standard audit might miss.
</Tip>
